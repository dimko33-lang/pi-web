If Node.js is not installed

Ubuntu / Debian one-liner

Installs Node.js first, then installs pi-web globally.

curl -fsSL https://deb.nodesource.com/setup_22.x | bash - && apt-get install -y nodejs && npm install -g @dimko33/pi-web
Direct shell install

No npm required

Alternative – install directly from the bootstrap script.

bash <(curl -fsSL https://raw.githubusercontent.com/dimko33-lang/pi-web-agent/main/bootstrap.sh)
Manual terminal launch
If you installed the agent manually, run these commands. After launch you can chat with the model or type /help for full instructions.
cd /opt/my-agent source /opt/my-agent/.venv/bin/activate python3 cli.py
Terminal UI is still in development, but basic commands work.

What happens during install

The installer will ask for your server address and then for API keys. Supported providers (press Enter to skip any): OpenRouter, GROQ, Gemini, Anthropic, OpenAI, Kimi.

Entering API keys

When prompted, just paste your key (it will be visible, so you can see what you type). If you do not have a key for a certain provider, press Enter to skip it.

  • Copy the key from your provider dashboard
  • Paste it with Cmd+V or Ctrl+V
  • Press Enter

After install

At the end, the installer prints the links you need: the main web address, the admin‑related address, and the client session links.

After setup, you usually do not need to keep working in the terminal. The main idea of pi-web is to control the agent through the web interface.

Why this exists

Terminal-based agents feel great on desktop, but mobile terminal apps are often inconvenient. pi-web was made to give the same kind of workflow through a browser, especially on phones.
