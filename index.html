What happens during install

The installer will ask for your server address and then for API keys. Supported providers (press Enter to skip any): OpenRouter, GROQ, Gemini, Anthropic, OpenAI, Kimi.

Entering API keys

When prompted, just paste your key (it will be visible, so you can see what you type). If you do not have a key for a certain provider, press Enter to skip it.

  • Copy the key from your provider dashboard
  • Paste it (the usual paste shortcut works)
  • Press Enter

After install

At the end, the installer prints the links you need: the main web address, the admin‑related address, and the client session links. After setup, you usually do not need to keep working in the terminal. The main idea of pi-web is to control the agent through the web interface.

Client panels (after install)

Once the installation finishes, the terminal will show several client panel links:

  • http://89.125.62.58/?-projects-pi-ru – main public panel
  • http://89.125.62.58/?-projects-pi-ru/1 – backup panel #1
  • http://89.125.62.58/?-projects-pi-ru/2, /3 … up to 5–6 independent client sessions

These links give end‑users access to the agent without admin rights. You can share them with different people – each session is independent. Just copy the desired link from the terminal output and open it in a browser.
The admin alias (with -admin-...) is separate and should be kept private.

Why this exists

Terminal-based agents feel great on desktop, but mobile terminal apps are often inconvenient. pi-web was made to give the same kind of workflow through a browser, especially on phones.
