Установка pi-web одной командой

Минимальная установка веб-агента прямо в браузере.

GitHub npm исходники сайта
Основная команда
npm install -g @dimko33/pi-web
Самый короткий путь установки.
Демо: клиентская панель
Только интерфейс агента. Никаких настроек сервера, выбора моделей или админ-функций.
http://78.17.36.139/?-projects-pi-ru
Открыть демо
Если Node.js не установлен

Ubuntu / Debian одной строкой

Устанавливает Node.js, затем глобально ставит pi-web.

curl -fsSL https://deb.nodesource.com/setup_22.x | bash - && apt-get install -y nodejs && npm install -g @dimko33/pi-web
Прямая установка через shell

Без npm

Альтернативный способ — установка напрямую из bootstrap-скрипта.

bash <(curl -fsSL https://raw.githubusercontent.com/dimko33-lang/pi-web-agent/main/bootstrap.sh)

Что происходит при установке

Установщик спросит адрес вашего сервера, а затем — API-ключи. Поддерживаемые провайдеры (Enter — пропустить): OpenRouter, GROQ, Gemini, Anthropic, OpenAI, Kimi.

Ввод API-ключей

Когда появится запрос — просто вставьте ключ (он будет виден, чтобы вы видели, что вводите). Если у вас нет ключа для конкретного провайдера, нажмите Enter.

  • Скопируйте ключ из личного кабинета провайдера
  • Вставьте его через Cmd+V или Ctrl+V
  • Нажмите Enter

После установки

В конце установщик выведет нужные ссылки: основной веб-адрес, адрес для администратора и клиентские ссылки.

После настройки вам обычно не нужно оставаться в терминале. Главная идея pi-web — управлять агентом через веб-интерфейс.

Зачем это нужно

Терминальные агенты удобны на десктопе, но на телефоне мобильные терминалы часто неудобны. pi-web создан, чтобы дать тот же рабочий процесс через браузер, особенно на телефонах.
